Контрольная работа №2. Анализ программ

 

1. Фрагменты должны быть не из тестов 2.5, а из любого раздела CPROG, в том числе тесты из «вопросы без ответов», либо из любых других источников.

2. Фрагменты должны представлять собой целостные алгоритмы, желательно в виде функций,  в которых используемый технологический прием НЕ ЯВЛЯЕТСЯ единственным, а входит как одна из составных частей.

3. Необходимо выделить СПК и прокомментировать его назначение в общем алгоритме.

4. Из одного фрагмента можно выделить несколько разных СПК.

5. Все СПК должны быть на разные темы (без повторений).

Пример

//------------------------------------------------------44-15.cpp
//------------------------------------------------- 8
 int F8(char c[])
 { int n,k,ns;
                 for (n=0,ns=0; c[n] !='\0'; n++) {
                 for (k=0; n-k !=0 && c[n+k] !='\0'; k++)    // 1 – результат цикла – место остановки
                                  if (c[n-k] != c[n+k]) break;     // 2 – сложение индексов –
                 if (k >=3) ns++;                                     // 3 - счетчик 
                 }
 return ns; }
 
Результат программы: количество симметричных фрагментов вида «abcba» длиной более 5 символов.
 
1.       Результат цикла – место его остановки – цикл сравнения останавливается, когда заканчивается совпадение. Результат – длина симметричного фпагмента
2.       Внутренний цикл использует симметричное сравнение символов относительного центрального (i+k, i-k)
3.       Счетчик ns во внешнем цикле определяет количество симметричных фрагментов
