"Английский" юмор в программировании

Многие анекдоты про программистов имеют смысл, понятный только при наличии знаний в определенных областях программирования.

Два стакана (анекдот)

Программист, ложась спать, ставит рядом два стакана. Полный - если захочет пить, пустой - если не захочет.
Комментарий:  с точки зрения единообразия представления данных предпочтительнее иметь "пустой" объект, имеющий вырожденное содержимое, нежели пустую ссылку (указатель).

class string{
    char *str;
public:   string(){ str=NULL; }             // Отсутствие стакана  - NULL указатель (1)
             string(){ str=strdup("");}     // Пустой стакан - строка из 0 символов (2)
             string(char c[]){
                    str=strdup(c);}             // Непустой стакан    (3)
             ~string(){ delete []str; }      // Деструктор на случаи  (2,3)
                };

Пословица

"Из файла байта не выкинешь".
Комментарий: в отличие от массива, где удаление элемента может быть выполнено путем сдвига "хвоста", в файле, несмотря на возможности позиционирования (и прямого доступа), такая процедура не применяется. Отчасти это связано с невозможностью "укоротить" уже существующий файл. Поэтому для удаления необходимо либо полностью переписать файл, либо отметить каким-нибудь образом элемент как "удаленный".
