Термины и определения

«Конечно, мама чтобы не ударить
лицом в грязь перед врачами, сама начала
изучать язык, на котором пишутся лекарства.
Для этого она собрала все рецепты,
склеила их в книжечку, и получился учебник».
Е. Чеповецкий. «Непоседа, Мякиш и Нетак».

Алгоритм – однозначное описание порядка (последовательности) выполнения действий из заданного набора, позволяющее получить требуемый результат за конечное число шагов.

Алгоритмически неразрешимые проблемы – задачи, для которых доказана невозможность разработки алгоритма ее решения (и, соответственно, создания программы).

Ассемблер - язык программирования на уровне компьютерной архитектуры, в котором система команд, способы адресации и машинные слова и их адреса обозначаются символическими именами.

Время жизни переменной - интервал времени работы программы, в течение которого переменная существует, для нее отведена память, и она может быть использована.

Двоичный файл - файл, в котором данные  представлены во внутренней форме, идентичной той, которая принята в памяти компьютера.

Динамический - элемент языка (свойство), изменяющееся во время выполнения программмы.

Интерпретация – непосредственное выполнение инструкций (команд, операторов)  формального языка над образами языковых объектов программы (обозначенных в программе именами).

Класс – описание структуры объекта и методов работы с ним,  – тип данных, определяемый программистом.

Компиляция – перевод текста программы с исходного формального языка на другой, более простой язык (например, машинный код).

Контекст - неявное окружение, к элементам которого можно обращаться непосредственно по именам объектов без указания каких-либо путей, префиксов, заголовков.

Логический - абстрактное представление, создаваемое из реального (физического), c помощью программных компонент.

Массив - множество объектов одного типа, доступ к которым осуществляется по номеру (индексу).

Метод – функция, работающая с объектом.

Наследование – неявное (по умолчанию) включение объекта одного класса в объект другого.

Неявный (по умолчанию) способ определения - свойство, определенное в языке при отсутствии явного определения каких-либо других свойств.

Область действия (видимости) переменной - та часть программы, в которой эта переменная может быть использована, то есть является доступной.

Объект – структура данных, содержащая описание свойств внешнего объекта программирования.

Объявление (декларация) - фрагмент программы, в котором  упоминается объект программирования (переменной, функции) и перечисляются его свойства, если он  в данной момент не доступен.

Определение - фрагмент программы, в котором полностью задано содержание объекта программирования (переменной, функции), и по которому происходит его трансляция во внутреннее представление.

Очередь  - последовательность элементов, включение в которую производится с одного, а исключение - с другого конца.

Переменная  – область памяти, в которой содержатся данные определенного типа. Имя переменной напрямую ассоциируется с ее адресом, содержимое памяти – со значением переменной.

Последовательный доступ - доступ к элементам последовательности в единственно возможном порядке, как правило, совпадающем с порядком их размещения (хранения).

Произвольный (прямой) доступ - доступ к элементам последовательности (множества) в любом порядке на основе операций выбора по порядковому номеру или позиционирования.

Прототип функции – заголовок функции со списком формальных параметров, заданных в виде абстрактных типов данных.

Рекурсия - называется способ построения объекта (понятия, системы, описание действия), в котором определение объекта включает аналогичные объекты (понятие, систему, действие) в виде составных его частей.

Связывание - процесс установления соответствия между объектами и их свойствами в программе на формальном языке (операции, операторы, данные) и элементами архитектуры компьютера (команды, адреса).

Ссылка – неявный указатель, имеющий синтаксис указуемого объекта (синоним). 

Статический  -   элемент языка (свойство), полностью определяемый при трансляции (и компоновке) и неизменный при ее выполнении.

Стек - последовательность, включение и исключение элементов в которую производится только с одного  конца.

Структура - множество объектов различных типов, доступ к которым осуществляется по имени.

Текстовый файл - файл, содержащий последовательность символов в сложившейся стандартной системе представления текста «символ-байт»

Тип данных - описание данных определенного вида, для которых известен их способ представления в памяти (формат),  размерность и диапазон значений, а также определен набор операций. Форма представления данных. «Идея» о представлении переменных определенного вида.

Транслятор – программа, которая позволяет системе, воспринимающей одни язык программирования, "понимать" программу, написанную на другом.

Трудоемкость программы (алгоритма) – зависимость количества массовых операций (сравнения, обмены, сдвиги, повторения цикла и т.п.) от объема (размерностей) обрабатываемых данных.

Указатель - переменная, содержимым которой является адрес другой переменной (переменная, содержащая информацию о расположении в памяти другой переменной).

Физический - реальный, имеющий место на самом деле на уровне компьютерной архитектуры.

Функция - логически и синтаксически завершенная часть программы с собственным программным интерфейсом.

Явный способ определения - свойство, определенное в языке "видимым" образом с помощью специального синтаксиса.
